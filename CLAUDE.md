# signet-eval

Claude Code PreToolUse hook policy enforcer. Deterministic policy evaluation — no NLP in the decision path.

## Quick Reference

```bash
pip install -e .
python3 -m pytest tests/ -v            # 40 contract tests
python3 -m signet_eval_tool --init      # Install default policy to ~/.signet/policy.yaml
echo '{"tool_name":"Bash","tool_input":{"command":"rm foo"}}' | python3 -m signet_eval_tool
```

## Structure

```
src/signet_eval_tool/
  signet_eval_tool.py    # All logic — policy engine, hook I/O, CLI
  __main__.py            # Entry point for python3 -m
tests/signet_eval_tool/
  contract_test.py       # 40 contract tests from Pact
```

## Key Design Decisions

- **No NLP in policy path.** Matching is regex + eval on structured data. No LLM judgment.
- **First-match-wins.** Rules evaluated in order. Predictable, auditable.
- **Fail-secure.** Malformed input, parse errors, any exception → deny.
- **Accept both formats.** `tool_input` (Claude Code hooks) and `parameters` (contract tests).
- **Default allow with explicit denials.** Block destructive ops, allow everything else.
- **60ms cold start.** Policy eval <1ms. Process startup is the bottleneck.

## Policy Format

YAML at `~/.signet/policy.yaml` (override via `SIGNET_POLICY_PATH` env var):

```yaml
version: 1
default_action: ALLOW
rules:
  - name: rule_name
    tool_pattern: "regex matching tool_name"
    conditions:
      - "Python expression evaluated with parameters/context in scope"
    action: ALLOW|DENY|ASK
    reason: "Human-readable reason"
```

Conditions use `eval()` with restricted builtins. `parameters` is a dot-accessible dict.
